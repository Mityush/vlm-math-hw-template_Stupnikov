## Анализ ошибок

Приведите 3 ошибки модели:

1. **Случайные предсказания** — используется случайно инициализированная tiny-модель без реального обучения; `parse_mc_answer` возвращает `None` на все примеры, поскольку генерируемые токены не соответствуют буквам A/B/C/D.
2. **Потеря пространственной информации в адаптере** — `AdaptiveAvgPool1d` усредняет патчи vision encoder, уничтожая пространственную структуру изображения; в результате модель одинаково обрабатывает график с линией вверху и линией внизу.
3. **Токенизация по пробелам** — `SimpleTokenizer` разбивает текст на слова-токены; русские флексии дают взрыв словаря (например, «треугольник», «треугольника», «треугольнику» — три разных токена), что ухудшает обобщение.

## Комментарии

Что оказалось самым сложным, что бы вы улучшили?

Сложнее всего оказалось корректно выстроить цепочку `processor → collate → model.forward`: нужно аккуратно согласовать позиции `<image>`-токенов в `input_ids` с визуальными эмбеддингами из адаптера, чтобы функция `merge_visual_embeddings` заменяла ровно нужные позиции. Отдельно пришлось продумать маскировку `labels` — `IGNORE_INDEX` для части-подсказки и реальные id только для ответа, чтобы loss считался правильно.

Что улучшил бы при наличии GPU:
- заменить tiny mock-бэкбоны на `google/vit-base-patch16-224` + `Qwen2-0.5B-Instruct`;
- обучить адаптер на `math_vqa_medium` с несколькими эпохами;
- добавить tiling с `num_tiles=4` для лучшего восприятия деталей (графики, геометрические схемы);
- измерить accuracy на `math_vqa_medium` dev-split как proxy-метрику качества.
